    全站逐篇複審計畫（維護者）¶

    本計畫用於系統性把每一篇文章過一遍：結構是否對、內容是否對、連結是否活、與 canonical 是否一致。

    與既有文件的關係

    文件 用途
    寫作規範 怎麼寫、A 級標準
    頁面章節規範 每頁應有哪些章節、應呈現什麼
    投資審查檢核表 內容對不對（五維 + P0～P3）
    內容覆蓋矩陣 等級與進度總覽
    reports/investment-review.md 複審發現與修正紀錄

    學員可略過本頁。

    一、目標與範圍¶

    項目 說明
    總篇數 136 篇 Markdown（含維護者頁）
    學員正文 118 篇（排除 8 維護規範 + 10 參考附錄中的純索引頁，依下方清單）
    完成定義 該篇勾選「結構 ✅ + 內容 ✅ + 連結 ✅」；若有修改則跑 CI 通過
    建議節奏 每批 8～12 篇 → 修正 → pytest + check_links → commit → 勾選

    二、單篇複審流程（約 15～25 分鐘／篇）¶

    每篇文章依序做：

    Step 1｜對類型（1 分鐘）¶

    對照 頁面章節規範 § 一，確認屬 A～H 哪一類。

    Step 2｜結構驗收（5 分鐘）¶

    • [ ] H1、## 本篇你會學到（例外頁除外）
    • [ ] 該類型必備章節齊全（看表要有手算、詞典要五表、案例要免責與反思）
    • [ ] ## 重點回顧 + 相關 連結
    • [ ] 教學頁有 ## 自我檢查（案例頁除外）
    • [ ] 無簡體字；英文術語僅首次括註（詞典除外）

    Step 3｜內容驗收（10 分鐘）¶

    對照 投資審查檢核表 的 A～E 區：

    • [ ] 數字：稅費、T+2、漲跌停、除息參考價、手算可重算
    • [ ] 用語：與詞典、canonical 一致
    • [ ] 風險：教學用、常見誤區、指標滯後／強勢股例外（技術面）
    • [ ] canonical：未重複展開權威章節（見 架構說明）
    • [ ] 案例：結論含「非投資建議」

    發現問題記 P0～P3；P0/P1 當批必修。

    Step 4｜連結與建置（改動後才做）¶

    uv run pytest tests/ -q
uv run python scripts/check_links.py --strict
uv run mkdocs build --strict

    Step 5｜登記¶

    • 在本頁下方對應列將 狀態 改為 ✅
    • 有投資內容爭議 → 寫入 reports/investment-review.md
    • 等級變動 → 更新 CONTENT-COVERAGE-MATRIX.md

    三、建議複審順序（12 批）¶

    原則：先 canonical／高風險數字 → 入門制度 → 看表看圖 → 模式風控 → 案例收尾 → 附錄與維護者頁。

    flowchart LR
    P0[批0 Canonical] --> P1[批1-2 入門]
    P1 --> P2[批3 看表]
    P2 --> P3[批4-5 看圖詞典]
    P3 --> P4[批6-8 分析模式風控]
    P4 --> P5[批9-10 老手人設案例]
    P5 --> P6[批11 附錄維護]
