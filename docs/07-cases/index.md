!!! warning "免責聲明"
    所有案例僅供教學示範，數據與情境為合成或匿名化整理，**不構成投資建議**，歷史不代表未來。

---

## 案例怎麼讀

每篇案例都依同一結構拆解，方便你套用到自己的交易：

```mermaid
flowchart LR
    A[背景] --> B[看到的表/圖]
    B --> C[推理步驟]
    C --> D[結論]
    D --> E[反思]
```

學會後，建議用 [研究流程](../09-advanced/research-workflow.md) 把自己的真實交易改寫成同樣格式複盤。

## 建議閱讀順序

- **完全新手**：先讀案例一、二、五，建立「資料 → 推理 → 結論」的習慣。
- **已選定模式**：直接挑上表對應你模式的案例。
- **老手複盤**：把每篇結論與 [老手陷阱](../09-advanced/veteran-pitfalls.md) 對照自檢。

## 自我檢查

??? question "1.（概念題）案例章的標準結構是什麼？"
    參考答案：**背景 → 資料 → 推理 → 結論（教學用）→ 反思**；非投資建議。

??? question "2.（判斷題）讀完案例就可以直接照著買？"
    參考答案：不行。結論皆標示**教學用**；要轉成自己的 [研究流程](../09-advanced/research-workflow.md)。

??? question "3.（情境題）完全新手建議先讀哪幾篇？"
    參考答案：案例**一、二、五**（營收轉折、鎚子均線、估值陷阱），建立推理習慣。
